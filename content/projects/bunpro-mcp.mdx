# Bunpro MCP Server

## Overview

Built to explore the Model Context Protocol, Bunpro MCP Server exposes Bunpro's Japanese-learning API to assistant workflows so I could ask grammar, vocabulary, reading, and study-progress questions using natural language.

## Challenge

The Bunpro API is undocumented, so every endpoint had to be reverse-engineered from client traffic. Delivering reliable answers meant anticipating every failure mode, building centralized error handling, and then validating every tool with MCP Inspector.

## Solution

FastAPI served as the async backbone, with MCP routes tailored for grammar, vocabulary, reading passages, stats, and reviews. Uvicorn/uv keeps performance tight while shared helpers enforce consistent error handling, and MCP Inspector runs keep each tool honest before it ships to a user-facing assistant.

## Results

The server now supports structured Bunpro interactions and doubles as a learning reference whenever I need to clarify Japanese study topics, demonstrating how MCP fundamentals, Python async practices, and resilient error handling work together in a learning-focused side project.
